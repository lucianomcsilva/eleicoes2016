#Convenções

Geral
- Variáveis devem ser sempre em portugues
- Variáveis devem possuir primeiros dois caracteres descrevendo o tipo.
   TP - Para variaveis de seleção (Ex.: Tipo de bloco - TP_BLOCO)
   IN - Variaveis Booleanas
   VL - Variaiveis que conteram valores. Pode ser texto ou numerico. (ex.: texto - Email, Numerico - Valor de um evento)
   DC - Variaiveis de texto longas, para descrição de algum elemento
   CD - Para variaveis do tipo codigo. Equivalente a ID em ingles.
   NM - Semelhante a DC. Porém nao tão longo.
   QT - Para uma variável inteira que se refere a quantidade de elementos (ex.: Quantidade de linhas)
   LC - para localização de objetos. Serve para caminhos no disco (ex.: /public_html/some/folder/) ou para enderecos na internet (ex.: www.somewebsite.com/some/path/page.html)
   LT - para lista de objetos
- Variáveis internas a uma classe devem ser sempre escrita em minusculas

Banco de dados
- Deve sempre ter os nomes das tabelas em portugues.
- Deve sempre ter os nomes das varíaveis em portugues.

Classes:
- Sempre iniciam com T, e possuir UpperCamelToe (ex.: TBlocoHtml);
- Os nomes das classes devem ser em portugues (TPagina);
- as váriaveis privadas devem iniciar com underscore (ex.: $_nm_pagina)


Funções avulsas :
- Sempre em ingles
- deve utilizar lowerCamelToe


Classes CSS:
- devem iniciar com ZS sempre que forem criada para os projetos da Zeetha
- devem sempre ser em inglês para faciliatar a internacionalização
- devem sempre ser minusculas e com espaçamento utilizando underscore (ex.: zs_)
